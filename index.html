header标签

HTML中的 header 标签用于定义文档或文档节的页眉部分。它通常包含网站或应用程序的标志、标题、导航栏和其他与页面整体相关的内容。
以下是 header 标签的一些主要作用：
1. 标识网站或应用程序： header 可以包含网站或应用程序的标志、标题和其他标识元素，帮助用户在页面上快速识别和识别所访问的网站或应用程序。
2. 导航栏：常见的做法是将网站的主导航栏放置在 header 中，以提供对网站不同部分的导航链接。这使得导航栏在整个页面中保持一致，并提供易于访问的导航功能。
3. 页面级别的标题或描述： header 中的文本内容可以用于呈现页面的主要标题或简短描述，使用户对页面内容有一个快速的了解。
4. 页眉内容：除了标志、标题和导航栏之外， header 还可以包含其他与页面整体相关的内容，如联系信息、搜索框、社交媒体链接等。
总的来说， header> 标签提供了一个容器，用于将与页面整体相关的内容组织在一起，并帮助用户快速识别和导航到页面的不同部分。它在构建语义化的HTML结构时起到了重要的作用。

section标签

HTML中的 section 标签用于定义文档中的一个独立的部分或区块。它通常表示文档的逻辑或主题上的分割，可以将相关内容组织在一起。
以下是 section 标签的一些主要作用：
1. 逻辑分割： section 标签用于将文档内容分割成不同的逻辑部分。每个 section 可以包含独立的主题、主要内容或功能模块，使得文档的结构更加清晰和有组织。
2. 内容组织： section 可以用于组织相关的内容，使其在视觉上或语义上形成一个完整的区块。例如，一个新闻网站可以使用 section 将每篇新闻文章包装在一个独立的区块中。
3. 标题和子标题： section 可以包含一个标题或子标题，用于描述该部分的内容或主题。这有助于读者快速了解该部分的主要内容。
4. 样式和样式隔离：通过给 section 添加class或ID，可以为每个部分应用特定的样式或脚本。这样可以更好地控制和定制每个部分的外观和行为。
5. 语义化的HTML： section 标签的使用有助于构建语义化的HTML结构，提高文档的可访问性和可理解性。它使得文档结构更加清晰，并为搜索引擎和辅助技术提供更准确的信息。
总的来说， section 标签用于划分和组织文档内容，提供更好的结构和语义化，使得页面的逻辑更清晰，并且可以更好地控制和样式化各个独立的部分。

div标签

HTML中的 div 标签是一个通用的容器元素，用于组合其他HTML元素或内容，并将它们作为一个整体进行样式化或操作。
以下是 div 标签的一些主要作用：
1. 分组内容： div 标签可以用于将相关的HTML元素或内容组合在一起，并将它们作为一个逻辑组进行处理。它提供了一种分组元素的方式，使得可以将一组相关的元素包装在一个容器中。
2. 样式化和布局： div 标签是CSS样式化和布局的主要工具之一。通过为 div 添加class或ID，可以为其应用特定的样式规则，控制其外观、位置和布局。
3. JavaScript操作： div 标签可以用作JavaScript操作的目标容器。通过选择和操作 div 元素，可以动态地添加、修改或删除其中的内容，实现与用户交互和动态页面更新相关的功能。
4. 语义化的HTML：尽管 div 本身没有明确的语义，但它在构建语义化的HTML结构时起到了重要的作用。通过将相关的内容包装在 div 中，可以提高HTML文档的可读性和可维护性。
5. 占位符： div 标签可以用作占位符，为将来的内容预留空间。在页面开发过程中，可以使用 div 作为临时容器，以便在后续的开发阶段填充实际内容。
总的来说， div标签是一个通用的容器元素，用于组合和包装其他HTML元素或内容。它在样式化、布局、JavaScript操作和语义化的HTML结构方面发挥着重要作用，并提供了一种灵活的方式来组织和处理页面上的内容。

HTML中的 ul 标签用于创建无序列表（unordered list）。无序列表是一种用于展示项目列表的标记方式，其中的项目没有特定的顺序或层次关系。

3. 分组相关内容：无序列表适用于将相关的项目进行分组显示。例如，在一个产品特点列表中，每个特点可以作为一个列表项进行展示。
4. 语义化的HTML：使用 ul 标签可以提高HTML文档的语义化。它明确地标识出包含无序列表的部分，使得搜索引擎和辅助技术能够更好地理解和解释列表的结构和含义。
5. 样式化和布局：通过为 ul 和 li 标签添加class或ID，可以为无序列表应用特定的样式规则，以实现自定义的外观和布局。
总的来说， ul 标签用于创建无序列表，其中的项目没有特定的顺序或层次关系。它在组织和展示项目列表、语义化的HTML结构以及样式化和布局方面起到了重要的作用。

算法数据结构

第一章 概述

1.1 算法复杂度

前言

算法复杂度旨在计算在输入数据量 N 的情况下，算法的「时间使用」和「空间使用」情况；体现算法运行使用的时间和空间随「数据大小 N 」而增大的速度。

算法复杂度主要可从 时间 、空间 两个角度评价：

时间： 假设各操作的运行时间为固定常数，统计算法运行的「计算操作的数量」 ，以代表算法运行所需时间； 空间： 统计在最差情况下，算法运行所需使用的「最大空间」； 「输入数据大小 N 」指算法处理的输入数据量；根据不同算法，具有不同定义，例如：

排序算法： N 代表需要排序的元素数量； 搜索算法： N 代表搜索范围的元素总数，例如数组大小、矩阵大小、二叉树节点数、图节点和边数等；

1.1.1 时间复杂度

概念定义

根据定义，时间复杂度指输入数据大小为 N 时，算法运行所需花费的时间。需要注意：统计的是算法的「计算操作数量」，而不是「运行的绝对时间」。计算操作数量和运行绝对时间呈正相关关系，并不相等。算法运行时间受到「编程语言 、计算机处理器速度、运行环境」等多种因素影响。例如，同样的算法使用 Python 或 C++ 实现、使用 CPU 或 GPU 、使用本地 IDE 或力扣平台提交，运行时间都不同。 体现的是计算操作随数据大小 N 变化时的变化情况。假设算法运行总共需要「 1 次操作」、「 100 次操作」，此两情况的时间复杂度都为常数级O(1) ；需要「 N 次操作」、「 100N 次操作」的时间复杂度都为 O(N) 。

符号表示

根据输入数据的特点，时间复杂度具有「最差」、「平均」、「最佳」三种情况，分别使用 O , Θ , Ω 三种符号表示。以下借助一个查找算法的示例题目帮助理解。

  • 最佳情况 Ω(1) ： nums = [7, a, b, c, ...] ，即当数组首个数字为 7 时，无论 nums 有多少元素，线性查找的循环次数都为 1 次；
  • 最差情况 O(N) ：nums = [a, b, c, ...]且 nums 中所有数字都不为 7 ，此时线性查找会遍历整个数组，循环 N 次；
  • 平均情况 Θ ： 需要考虑输入数据的分布情况，计算所有数据情况下的平均时间复杂度；例如本题目，需要考虑数组长度、数组元素的取值范围等；

11.1 曲线

曲线 Curve ，也称为 样条 ，用于描绘平滑的曲线和形状

一般分为两种：

  • 贝塞尔曲线
  • NURBS曲线

添加一条曲线，演示：

  • 添加|曲线|贝塞尔曲线
  • 按 7 ，进入顶视图
  • 按 TAB ，编辑曲线
  • 选择移动工具，移动 控制点 或 控制柄
  • 按 TAB ，退出编辑

接下来，可以将曲线转换为网格，或其他用途。

在大纲视图中，曲线有自己独特的图标

11.2 控制点

添加一条贝塞尔曲线，编辑。

选中控制点时，显示手柄。

  • 控制点
  • 控制柄

两种操作方式：

1 框选，移动或 g 键

2 调整

手柄可以伸长、缩短，会影响曲线的形状。

（4个点来定义一条曲线，2端点、2参考点）

11.3 编辑曲线

1 添加控制点

  • 细分 ：选中两个控制点，右键，细分
  • 挤出 ：选中端点，按 E 键

2 删除控制点

  • 融并顶点 ：选中控制点，按 X 键

另外，按 A 键，全选所有控制点

选中一个控制柄，按 V 键，控制柄类型

  • 自由，自由调节
  • 对齐，左右同步
  • 矢量，直接连接
  • 自动，自动平滑

左右控制柄可以单独设置
